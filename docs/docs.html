How do I set up WhiteRoom?

WhiteRoom works by redirecting your LLM API calls through the WhiteRoom proxy. Your agent framework continues to run unchanged — governance is applied transparently.

Step 01

Set one variable

Point your LLM base URL at WhiteRoom. Your existing API key stays the same.

Step 02

Run your agent

No code changes needed. WhiteRoom intercepts calls, applies governance, and forwards to the LLM.

Anthropic (Claude)

Terminal export ANTHROPIC_BASE_URL=https://proxy.whiteroom.tech

OpenAI

Terminal export OPENAI_BASE_URL=https://proxy.whiteroom.tech/v1
Bring your own key. WhiteRoom uses your existing API key — it never stores or accesses your credentials. The proxy intercepts requests to apply governance and forwards them to your provider.

Which frameworks does WhiteRoom support?

WhiteRoom is framework-agnostic. It works at the API level, so any framework that calls an LLM API is automatically governed once you set the base URL.

LangChain

Python # Set the env variable, then use LangChain as normal import os os.environ["OPENAI_BASE_URL"] = "https://proxy.whiteroom.tech/v1" from langchain_openai import ChatOpenAI llm = ChatOpenAI(model="gpt-4o") # All calls are now governed by WhiteRoom

CrewAI

Python # Set before importing CrewAI import os os.environ["OPENAI_BASE_URL"] = "https://proxy.whiteroom.tech/v1" from crewai import Agent, Task, Crew # Agents are automatically governed

Direct API (Python)

Python import anthropic client = anthropic.Anthropic( base_url="https://proxy.whiteroom.tech" ) # Your existing API key is used automatically

What governance does WhiteRoom apply?

Once connected, WhiteRoom automatically applies three governance controls to every agent session.

Session boundaries

Each agent session is scoped with configurable limits. When a session approaches its boundary, WhiteRoom manages the transition automatically — keeping agents focused and preventing unbounded token accumulation.

Token optimization

WhiteRoom's proprietary optimization engine automatically reduces token usage — typically by 40–80% depending on the workload. See the benchmark for detailed A/B test results.

Audit trail

Every agent action, token usage event, and session boundary is logged to a hash-chained, tamper-evident audit trail. Each entry is cryptographically linked to the previous one, making any modification or deletion detectable. The audit trail streams in real time.

API reference

WhiteRoom exposes two endpoints. The governance API manages fleets, agents, watches, and handovers. The proxy endpoint forwards LLM requests through the governance layer.

POST /api/white-room

All governance actions use a single endpoint with an action field in the JSON body. Auth via x-api-key or Authorization: Bearer header.

ActionRequired fieldsDescription
register_agentagent_idRegister an agent in a fleet and receive a fleet token
start_watchagent_idStart a bounded work session
complete_taskagent_id, task_nameRecord a completed task with time and token usage
check_watchagent_idCheck remaining time and whether handover is needed
initiate_handoveragent_id, to_agentHand off context from outgoing to incoming agent
fleet_report—Full fleet status, totals, compliance, and recent audit
audit_log—Query the tamper-evident audit trail
token_loginfleet_tokenAuthenticate with a fleet token (dashboard login)

POST /v1/messages

Anthropic Messages API proxy. Sends x-api-key and receives full governance: watch enforcement, automatic handovers, and audit logging. Governance blocks return 429 with a whiteroom object.

SDK & CLI

For programmatic governance control, use the TypeScript SDK. For terminal-based fleet management, use the CLI.

SDK

npm install @whiteroom-ai/sdk
import { WhiteRoomClient } from "@whiteroom-ai/sdk";

const wr = new WhiteRoomClient({
  baseUrl: "https://proxy.whiteroom.tech",
  apiKey: process.env.WHITEROOM_API_KEY!,
});

await wr.registerAgent("agent-1", "default");
await wr.startWatch("agent-1");
const status = await wr.checkWatch("agent-1");
console.log(status);

CLI

npx @whiteroom-ai/cli health --url https://proxy.whiteroom.tech
npx @whiteroom-ai/cli register --url https://proxy.whiteroom.tech --key $API_KEY agent-1
npx @whiteroom-ai/cli start --url https://proxy.whiteroom.tech --key $API_KEY agent-1
npx @whiteroom-ai/cli check --url https://proxy.whiteroom.tech --key $API_KEY agent-1
npx @whiteroom-ai/cli report --url https://proxy.whiteroom.tech --key $API_KEY

Set WHITEROOM_URL and WHITEROOM_API_KEY environment variables to skip the flags.
